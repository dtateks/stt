## Research Context
> Actionable knowledge from librarian and oracle that executors need for correct implementation.

### Library Deep Dives (from librarian)
- **WKWebView Browser Helper ownership**: continuous canvas rendering/compositing work is carried by the WebKit GPU / Browser Helper process, so reducing visible-state redraw work and per-frame canvas allocation is a first-order thermal fix, not a micro-optimization. Visible overlay windows do not benefit from background-tab throttling. — implement by removing wasted work rather than relying on hidden/offscreen heuristics (source: librarian TYPE D artifact `artifacts/libs/wkwebview-hud-cpu-ram-spike_23.41_01-04-2026.md`, `artifacts/libs/wkwebview-hud-cpu-second-pass_00.01_02-04-2026.md`)
- **macOS window-config conflict**: upstream evidence is conflicted — `resizable(false)` has a known excessive-CPU report on macOS (`tauri#11308`), while borderless + resizable windows also have a known high-CPU path on macOS 15 (`tao#1191`). Do not blindly flip one flag. Choose the safest runtime/config change for this repo’s exact Tauri/tao/NSPanel path and preserve the fixed-size HUD contract. — source: librarian TYPE D artifact `artifacts/libs/wkwebview-hud-cpu-second-pass_00.01_02-04-2026.md`
- **WebAudio lifecycle**: `AudioContext` / analyser work is expected for live capture, but repeated teardown/rebuild amplifies native-resource churn. Prefer reuse within a visible session or a deterministic teardown barrier over rapid rebuild loops. — source: librarian TYPE D artifact `artifacts/libs/wkwebview-hud-cpu-ram-spike_23.41_01-04-2026.md`

### Architecture Rationale (from oracle)
- **Preserve UX first**: the current polished HUD UX is not inherently the bug. Fix invisible wasted work first (render gating, allocation reuse, lifecycle reuse, hostile macOS window config). Only if that is insufficient may the executor apply minor waveform-only adaptive tuning; do not degrade transcript flow, transparency, placement, or always-on-top behavior. — source: oracle task `ses_2b5fc1107ffeKPCZqFtiWQ1Y6u`, interview

## Design Decisions
- **Preserve the existing HUD contract**: no opaque HUD, no hide-during-processing behavior, no new user-facing settings, and no removal of fullscreen/always-on-top overlay behavior.
- **Fix waste before compromise**: Wave 1 targets three invisible root-cause classes in parallel — render/compositor waste, audio/session churn, and macOS runtime/config hot spots.
- **Waveform fallback is tightly constrained**: if invisible fixes alone cannot hit the thermal target, only minor adaptive waveform tuning is allowed, and only for waveform motion/detail outside active speech.
- **Executor build evidence stays in reports**: each implementation task must run `npm run build` and report the result, but orchestrator acceptance commands stay non-destructive and side-effect-free.

## Pattern Conflicts
- **macOS HUD CPU reports**: chose a measured, version-aware macOS fix over an automatic `resizable` flip because upstream evidence says both `resizable:false` and borderless + `resizable:true` can be pathological in different macOS/Tauri paths. Task 03 owns that resolution.
